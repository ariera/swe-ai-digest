GLM-5.1: Towards Long-Horizon Tasks (2026-04-07)

For engineers tracking open-weight models and on-device/API alternatives to closed frontier models, GLM-5.1 represents a significant step in open-model quality and autonomous code correction, and its MIT license makes it particularly relevant for production deployments.

Anthropic's Project Glasswing - restricting Claude Mythos to security researchers - sounds necessary to me (2026-04-07)

Simon Willison covers Anthropic's decision not to publicly release Claude Mythos, their most capable model, due to its unprecedented autonomous exploit-development abilities. Instead, Anthropic launched Project Glasswing — a restricted preview program for trusted partners including AWS, Apple, Microsoft, and Google — backed by $100M in usage credits. Willison cites concrete evidence: Mythos found vulnerabilities in every major OS and browser, wrote a four-vulnerability browser exploit chain, and achieved an exploit success rate of 181/~200 attempts on Firefox vs near-0% for the previous Claude Opus 4.6. Maintainers of Linux, curl, and HAProxy all confirm a sudden shift from AI-generated noise to high-quality, real reports. Willison concludes the caution is warranted and represents an industry-wide security reckoning engineers should track closely.

Google AI Edge Gallery (2026-04-06)

Simon Willison reviews Google's AI Edge Gallery, an official iPhone app for running Gemma 4 models (E2B and E4B) locally on-device. He finds the E2B model — a 2.54GB download — fast and genuinely useful, with multimodal capabilities including image Q&A and 30-second audio transcription. The app also demonstrates tool-calling via eight interactive widget 'skills.' Willison notes this is the first time a major model vendor has shipped an official on-device iOS app for their open models. Limitations include no persistent conversation logs. For software engineers, this signals that on-device inference for capable vision and audio models is moving from experimental to consumer-ready, with direct implications for privacy-sensitive and offline use cases.

Eight years of wanting, three months of building with AI (2026-04-05)

Willison surfaces Lalit Maganti's detailed account of building syntaqlite — a production-quality SQLite parser and linter — over three months with heavy Claude Code assistance, after eight years of procrastination. The key finding is that AI dramatically lowered the activation energy to start, converting abstract design anxiety into concrete prototype work. However, the first AI-generated prototype had to be discarded: AI deferred architectural decisions, created an incoherent high-level structure, and made cheap refactoring a trap for postponing real design thinking. The successful second attempt required deep human-in-the-loop judgment on architecture. Maganti's central insight — that AI excels at objectively verifiable implementation tasks but is 'between unhelpful and harmful' when the task lacks a checkable right answer — is directly applicable to any engineer using coding agents on complex systems.

Vulnerability Research Is Cooked (2026-04-03)

Willison highlights Thomas Ptacek's analysis arguing that frontier LLM coding agents will fundamentally reshape vulnerability research within months. Ptacek's argument is structural: LLMs are uniquely suited to exploit development because they encode vast correlations across source code, have built-in knowledge of all documented bug classes (type confusion, integer mishandling, stale pointers), and can run testable exploit trials indefinitely without fatigue. The economics of exploit development will shift accordingly. Ptacek predicts a step-function change rather than gradual improvement. For software engineers, this means the threat model for their codebases is changing faster than patching cycles, and that security tooling built on agent architectures will soon outpace human-speed vulnerability research.

Chief Scientist at ThoughtWorks, author of Refactoring and Patterns of Enterprise Application Architecture, and original signatory of the Agile Manifesto.

Fragments: April 2 (2026-04-02)

Martin Fowler's fragments post synthesizes several AI-adjacent ideas. He introduces a three-layer debt model (technical, cognitive, intent debt) as a framework for understanding how AI-generated code erodes team comprehension. He covers the 'System 3' cognitive model — treating LLMs as a third cognitive system that introduces 'cognitive surrender' when used uncritically. Most practically, Fowler endorses the argument that if agents make coding free, verification becomes the scarce and expensive resource, requiring organizational restructuring: fewer engineers writing code, more defining acceptance criteria and test harnesses. He also surveys the open question of whether source code itself has a future. This post provides a coherent theoretical framework for the practical observations made by Beck, Willison, and others this week.

Harness engineering for coding agent users (2026-04-02)

Martin Fowler's site publishes Birgitta Böckeler's fully developed mental model for 'harness engineering' — the practice of structuring codebases, test suites, and development environments specifically to enable coding agents to work effectively. The article argues that driving agents well requires deliberate investment in the scaffolding around agents (CLAUDE.md files, automated test harnesses, clear interfaces) rather than just prompting skill. This is one of the more concrete, actionable frameworks published this week for software engineers who want to move beyond ad-hoc agent use toward systematic agentic development. It directly addresses the verification bottleneck identified by Fowler's fragments post and the architecture problems Maganti encountered.

Absurd In Production (2026-04-04)

Armin Ronacher reports on five months of running Absurd — a durable execution system built entirely on Postgres — in production at Earendil, primarily for agent workflows. He confirms the checkpoint-based model held up: each LLM call iteration is a step; if the process dies mid-agent-loop, it replays from the last checkpoint. New additions include decomposed steps for conditional logic, task result inspection for parent/child agent orchestration, a CLI tool (absurdctl), and a web dashboard (Habitat). Ronacher also notes the system is used for cron deduplication and background processing. The post is directly relevant to engineers building durable agent architectures, offering a concrete, production-validated alternative to heavyweight workflow engines like Temporal, at a fraction of the SDK complexity (~1,400–1,900 lines vs. 170,000).
